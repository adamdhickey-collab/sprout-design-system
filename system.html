06 / Themes

Light & dark

Sprout supports two display themes that adapt the same component to its surface. Light Mode pairs a bright background with deep text and color for clarity in daylight environments. Dark Mode reverses it with a deep background and high‑contrast text for low‑light viewing and conserves screen energy.

SDS · Token naming hierarchy

sds · Component* · Foundation · Property* · Tier · Modifier*

* denotes levels that can be included for a more detailed mapping of the token. Example: sds-color-background-brand-hover resolves at the global level; sds-button-color-background-primary-hover resolves at the component level.

07 / Elevation

Shadows

Elevation creates a sense of hierarchy, depth, and importance. Use it to signal which elements are interactive, which are in the foreground, and which are secondary. Sprout ships three steps that cover most surfaces.

shadow-sm Hairline. Pressed buttons, subtle borders.
shadow-md Cards, popovers, dropdowns, toasts.
shadow-lg Modals, dialogs, full overlays.

09 / Illustration

Illustration & diagrams

Illustration is a functional brand asset, used only when photography won't suffice — abstract concepts, infographics, process diagrams. There is one illustration style across the brand, built from the same primary + secondary palette as everything else in this system; the guidelines are explicit that we never invent our own tints. Diagrams are the functional half of the chapter, fully specified below: line weights, fill treatments, and a worked process-flow example.

Line weights · fixed 1:2 ratio, scale uniformly

.75pt / 1pxDetail line

1.5pt / 2pxPrimary line

Alt text, once the real asset lands. Describe the scene's subject and action — what it shows and why — never "illustration of…" or the filename. If the illustration sits next to text that already says the same thing, mark it decorative with alt="" instead of describing it twice.

Character illustration · asset dependency

Not built here. The brand's people and object illustrations (the padlock, the corn, the test tube, every human figure) are hand-drawn in one approved style, with a dedicated skin-tone and hair-color palette (guidelines p.104) reserved for that use only — not brand colors, and not usable anywhere else. No source files for that style exist in this repo, and the guidelines explicitly warn against off-style substitutes, so this system does not attempt hand-coded stand-ins. Use the illustration placeholder above to hold the layout until the approved asset library lands, the same way brand photography is standing in for a licensed library today.

10 / Data visualization

Data visualization

Data is a functional element first — the brand's own rule is that color differentiates data points, but functionality always takes precedent over color, and a chart should never lean on color alone. Series colors follow a fixed order, never a cycled or arbitrary hue rotation, and every chart is either functional (the everyday, template-driven default) or expressive (a bolder treatment earned only in high-craft moments) — never a blend of both.

Validated, not just documented. Checked against the accepted color-vision-deficiency and contrast rules a chart palette needs: steps 1–3 separate cleanly, but the 4–6 tint pair (Leaf 70% vs. Deep 70%) sits close enough that even full-color vision struggles to tell them apart at a glance. Treat 4–6 as a last resort, not a default — reach for one of the secondary sets above before doubling into tints, and always pair every series with a direct value label so color is never the only thing carrying the data, per the brand's own accessibility note.

Functional vs. expressive · two different jobs, never mixed

Functional data

The everyday default — presentations, dashboards, functional documents. Built from a template and used as it comes: square corners (rounded corners aren't replicable across every platform that renders this data), clear labels, no unnecessary styling. This is what almost every chart in this system should be.

Expressive data

880,000

Reserved for high-craft moments — an annual report, a brochure — where design craft is earned. Rounded corners and a bolder, more bespoke label treatment are permitted here only; a dark canvas with the primary palette (black, white, Leaf Green, Soft Green) carries a single bold statistic further than a template chart could.

Infographics · proportion by repetition

person person person person person person person person person person

3 in 10 — a single repeated glyph, colored by proportion. The brand treats infographics as functional, not illustration: they may borrow the graphic language (the leaf, a single icon), but the job is still to communicate a number clearly, not to decorate.

Don'ts. Never round the corners on functional/template data — it isn't replicable everywhere this data renders. Never let color alone carry the difference between series; a legend or a direct label always accompanies it. Never reach for a secondary colorway "sporadically" — it's an intentional accessibility or brand-architecture choice, not decoration. And never build a chart that looks like data but communicates nothing of importance — if the content next to it is really a bullet list, use a bullet list.
